## Issue Guidelines

### Raising an Issue

When raising an issue, please provide detailed information including:

- Description of the problem or enhancement request.
- Steps to reproduce (if applicable).
- Any relevant screenshots or code snippets.

### Picking an Issue

To pick an issue to work on:

1. Browse the list of open issues.
2. Look for issues labeled with `help wanted` or `good first issue` if you're new to the project.
3. Assign yourself to the issue before starting work on it to avoid duplication of efforts.

### Resolving an Issue

When resolving an issue:

1. Assign yourself to the issue to indicate that you're working on it.
2. Fork the repository and create a branch for your changes.
3. Make the necessary changes and submit a pull request.
4. Reference the related issue in your pull request using the appropriate keywords (e.g., `Fixes #issue-number`, `Closes #issue-number`).
5. Await feedback and address any requested changes.

## Code Review

All pull requests will undergo a code review process. Please be responsive to feedback and make necessary changes promptly.

## Thank You!

Your contributions are greatly appreciated. Together, we can make Battleship even better! If you have any questions or need further assistance, feel free to reach out to us.
